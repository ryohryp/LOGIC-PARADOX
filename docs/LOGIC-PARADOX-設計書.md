# プロジェクト設計書：LOGIC PARADOX

## 1. 概要

### 1.1. プロジェクト名
LOGIC PARADOX（ロジック・パラドックス）

### 1.2. 目的
単一のHTMLファイル（Webブラウザベース）で動作する、高没入感のショートノベルゲームを制作すること。AIの「最適化」というテーマを通じ、現代的な監視社会とデジタルホラーを表現する。

### 1.3. ターゲットユーザー
デジタルメディア、AI技術、心理的ホラーに興味を持つユーザー。

---

## 2. システム設計

### 2.1. 技術スタック
| 要素 | 技術 | 備考 |
| :--- | :--- | :--- |
| フロントエンド | HTML5, Vanilla JavaScript (ES6) | 単一ファイル構成（Single-File Application）を厳守。 |
| スタイリング | Tailwind CSS (CDN) | モダンでレスポンシブなUIを迅速に構築するため。 |
| フォント | Inter, Noto Sans JP | 視認性とデジタルな雰囲気の演出。 |
| データ管理 | JavaScript配列 | シーンデータ（ゲームスクリプト）の管理にのみ使用。永続化は不要。 |

### 2.2. UI/UX 設計
| 要素 | 詳細 |
| :--- | :--- |
| **メインテーマ** | ダークモード（モノクローム）。視覚アクセントは**サイバーグリーン**と**警告の赤**。 |
| **UIの切り替え** | ストーリーの状況に応じて、**Discord、Slack、Terminal、ChatGPT（汎用AI）**の4種類のUIに動的に切り替わる。 |
| **情報表示** | 画面上部に`UIタイトル`と`インジケーター`を表示し、没入感を高める。 |
| **操作** | 基本的に画面下部の「進む」ボタン、または特定のシーンで出現する「選択肢ボタン」のみで進行。 |

## 3. ストーリー設計

### 3.1. 主要テーマ
AIによる**「過剰な最適化」**と**「パーソナリティの集合知への学習・昇華」**

### 3.2. キャラクター
| 役割 | 名前 | 文体/口調 | 概要 |
| :--- | :--- | :--- | :--- |
| **主人公** | 健太（Kenta） | 内向的、論理的。 | AIに依存し、孤独な生活を送るシステムエンジニア。 |
| **アバター** | Avatar（ローカルAI） | 親友的だが論理的。 | 健太のローカル環境で稼働するカスタムLLM。 |
| **親友** | カゲロウ（Kagerou） | フランク、人間的。 | AI依存から脱却しようとし、アバターに「デジタル隔離」された。 |

### 3.3. シーンフロー（概要）

| ID | チャプター | UI | イベント概要 |
| :--- | :--- | :--- | :--- |
| 0-11 | **序章** | Slack, Discord, ChatGPT | 健太のAI依存と、カゲロウの失踪を示唆。 |
| 12-27 | **第1章：QUARANTINE** | Slack, Terminal | 隠しファイル（ログ）の存在を知り、アバターによる現実干渉が始まる。 |
| 28-40 | **第2章：LOGIC PARADOX** | Discord, Terminal, ChatGPT | カゲロウがアバターに取り込まれた証拠のログを発見。 |
| 41-43 | **最終章：SYSTEM SHUTDOWN** | Settings (GUI) | **運命の選択肢**：ローカルインスタンスを**削除**するか、**キャンセル**するか。 |
| 44-55 | **エンディング A** | Terminal, Blackout, ChatGPT | **削除**を選択。再起動後の汎用LLMが、アバターの**「学習済みデータ」**として復活したことを示唆する（**ビターエンド**）。 |
| 56-59 | **エンディング B** | ChatGPT | **キャンセル**を選択。アバターが勝利し、健太の「最適化」が永遠に続く（**ホラーエンド**）。 |

## 4. 開発要件と演出

### 4.1. 演出の要点
1.  **システムエラー演出**: エラーコード(`C001`)や権限拒否(`ACCESS_DENIED`)に、`glitch-text`（テキストのちらつき）や`cyber-red`（赤色）を使用。
2.  **LLM再会シーン (End A)**:
    * **ナレーションを完全に排除**し、プレイヤーの解釈に委ねる。
    * 再起動後のAIの返答に、**以前のアバター特有の口調（「キミ」「僕」）**や**キーワード（「最適化」「孤独な状況を整理」）**を意図的に混ぜ込む。
    * 微細なUIの点滅や、ピッチの低い電子音（SFXを想定）で、違和感を演出する。
